# Kafka 

- 本质上是个消息队列，也可以支持流处理

### 架构

主要分为 3 个 component，producer, consumer, and Kafka cluster

主要还有 4 个 API

(1) producer API

(2) consumer API

(3) stream API

(4) connector API


其他术语：
- Topic：用于分类的消息
- Broker： 用于复制消息的 主机服务器？？？
- Partition：比 Topic 更小的一个概念，一个 Topic 里面有若干个 partition，用来提高消息处理的效率
